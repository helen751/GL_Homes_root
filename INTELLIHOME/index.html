INTELLIGENT CORE

Our system uses sensor_readings table to capture temperature, humidity, light, smoke. Device_commands queue controls bulbs and buzzer. Emergency_events logs smoke incidents. Alerts_log tracks severity.

Sensor Fusion

Distance, smoke, humidity – all indexed for fast retrieval.

Command Queue

bulb1, bulb2, buzzer, update_threshold – executed reliably.

Emergency Logic

Smoke > threshold triggers emergency_flag and buzzer_status.

DEVICE STATUS SNAPSHOT

Current state of bulbs, buzzer, auto modes, thresholds stored in device_status table. Updated in real time.

Bulb1 / Bulb2

Manual or auto controlled via light_level & motion_distance thresholds.

Buzzer & Emergency

emergency_flag =1 activates buzzer; acknowledge clears.

Auto Modes

auto_light_enabled, auto_motion_enabled, smoke_threshold configurable via command.
